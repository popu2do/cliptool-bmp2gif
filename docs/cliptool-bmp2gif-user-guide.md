# ClipTool 多图合成 GIF 使用指南

## 1. 这是什么工具

ClipTool 是一个 Windows 剪贴板图片转 GIF 工具。你在文件资源管理器里复制图片后，它会自动把图片追加到窗口里，方便你检查顺序、删减、重排，再把 GIF 写回剪贴板。

这意味着你最后不用先保存文件，再手动导入别的工具。最常见的动作还是：

1. 在资源管理器里选图并复制。
2. 在工具里确认顺序。
3. 点击生成。
4. 回到目标软件里直接 `Ctrl+V`。

## 2. 怎么启动

通常从启动器里点开这个工具即可。启动后会出现一个窗口，默认进入监听状态。

如果是直接运行可执行文件，也建议保持在工具自己的工作目录下启动，这样临时文件会落在工具自己的 `temp/` 目录里，排查时更容易找。

## 3. 基本使用流程

### 第一步：复制图片

在 Windows 文件资源管理器中选择一张或多张图片，按 `Ctrl+C`。

工具会把支持的图片追加到列表里。它不会替换你已经收集好的图片。

### 第二步：检查顺序

你会在窗口里看到缩略图。这里的顺序就是最后生成 GIF 的顺序。

如果顺序不对，可以直接拖拽缩略图调整。

### 第三步：继续追加

如果你还没复制齐，可以回到资源管理器继续复制别的图片。工具会继续追加，不会把之前的内容冲掉。

同一路径的图片会被去重，重复复制通常只会跳过，不会重复加入。

### 第四步：设置延迟

延迟控制每一帧停留多久。默认是 `500 ms`。

如果你只是做普通的指纹图对照，通常保持默认就够了。想让动图更快或更慢，再手动调整。

### 第五步：生成 GIF

点击“生成 GIF”或按 Enter。

生成完成后，GIF 会被写回剪贴板，你可以直接到目标软件里按 `Ctrl+V` 粘贴。

生成成功后，当前列表会自动清空，方便你开始下一轮采集。

## 4. 支持的图片

当前工具支持以下输入：

- `BMP`
- `PNG`
- `JPG`
- `JPEG`
- `RAW` 指纹图：`43808` 字节，`148x148`
- `BIN` 指纹图：`102400` 字节，`160x160`

注意：

- 只有被识别为支持格式的文件才会加入列表。
- 少于 2 张有效图片时，不能生成 GIF。
- 部分无效文件会被跳过，不会阻塞整个流程。

## 5. 你会最常用的操作

### 追加图片

从资源管理器再复制一批图片即可。工具会继续往后加。

### 删除单张图片

如果某一张不想要，直接在窗口里删掉它即可。只会移出当前列表，不会删除磁盘文件。

### 清空当前列表

如果这轮图片收集错了，直接清空重新来。

### 反转顺序

如果资源管理器里的选取顺序和你想要的帧顺序相反，可以先用反转，再做微调。

### 置顶

如果你要一边复制图片一边操作别的软件，可以把窗口置顶，减少来回切换。

## 6. 常见工作流示例

### 示例 1：一次复制齐

1. 在资源管理器里多选 5 张图。
2. `Ctrl+C`。
3. 看窗口里缩略图是否顺序正确。
4. 点击“生成 GIF”。
5. 到目标软件里 `Ctrl+V`。

### 示例 2：分几次复制

1. 先复制前 3 张。
2. 再复制后 2 张。
3. 如有需要拖拽调整顺序。
4. 生成 GIF。

### 示例 3：有重复图

1. 你不小心把同一批图复制了两次。
2. 工具会按路径去重，重复项通常会被跳过。
3. 继续确认剩下的帧顺序即可。

## 7. 注意事项

- 这是 Windows 工具，只适合 Windows 环境。
- 不是通用 GIF 编辑器，不支持裁剪、涂画、文字编辑这类动作。
- 生成成功后会清空列表，所以如果你还想保留当前列表，先别点生成。
- 生成失败时通常会保留当前列表，便于你修正后重试。
- 工具会清理自己的临时目录，正常情况下不用手工处理 `temp/`。
- 复制的是文件路径，不是把图片内容“存进项目里”。磁盘文件本身不会被修改。

## 8. 问题排查

### 8.1 复制了图片，但窗口没有新增

先检查这几件事：

1. 复制的是不是支持的图片格式。
2. 文件是不是已经被别的程序占用或损坏。
3. 复制动作是不是来自资源管理器的文件复制，而不是别的软件里的图片复制。

### 8.2 顺序和预期不一致

处理方式：

1. 先看窗口里现有顺序。
2. 必要时用“反转”。
3. 再通过拖拽做细调。

如果你在资源管理器里是多次复制，最后的顺序一定以窗口里当前显示顺序为准，不要依赖复制时的“感觉顺序”。

### 8.3 生成按钮不可用

通常是以下原因：

- 有效图片少于 2 张。
- 正在生成中。
- 当前列表里已经没有可用帧。

### 8.4 生成后粘贴不出来

先确认目标软件当前焦点在编辑区，再按 `Ctrl+V`。

如果还是不行，优先检查：

1. 生成是否成功。
2. 剪贴板是否被别的软件改写。
3. PowerShell 是否正常可用，因为工具会用它把 GIF 写回剪贴板。

### 8.5 退出后临时文件异常

工具启动和退出时都会清理临时目录。若中途被强制结束，可能会留下临时文件。重新启动一次通常就会整理掉。

## 9. 与启动器集成说明

这个工具给启动器接入时，重点不是复杂参数，而是把入口和工作目录配对好。

### 9.1 推荐接入方式

- 启动器直接指向 `cliptool.exe`。
- 启动时保持工具自己的工作目录不变，或显式设置到工具目录。
- 不要把启动器工作目录切到别的地方后再启动，否则 `temp/` 会跟着当前目录走，排查会变麻烦。

### 9.2 启动器侧需要关注的点

1. 入口是否指向当前版本的可执行文件。
2. 是否会重复拉起多个实例。
3. 更新或同步后，是否保留了工具目录下的相对路径结构。
4. 日志或报错是否能让测试同事看懂，而不是只返回一句“启动失败”。

### 9.3 对测试同事更友好的说明

如果你是通过启动器打开这个工具，日常只需要记住：

1. 先复制图，再看窗口。
2. 顺序不对就拖。
3. 点生成后，去目标软件直接粘贴。

启动器只负责把工具拉起来，不改变这个使用习惯。

## 10. 你应该优先记住的几条

1. 先在资源管理器里复制图片。
2. 工具窗口里调整顺序。
3. 至少 2 张有效图片才能生成。
4. 生成成功后会清空列表，并把 GIF 放到剪贴板。
5. 这是 Windows 专用工具。
