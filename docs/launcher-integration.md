# 启动器集成说明

这份说明给启动器维护者看，重点是把 `cliptool-bmp2gif` 接到统一入口时，怎样减少误配和排查成本。

## 1. 集成目标

- 从启动器里一键拉起工具。
- 保持工具自己的行为不变。
- 让普通测试同事不需要接触命令行。

## 2. 推荐配置

### 2.1 入口

优先直接调用构建产物 `cliptool.exe`。

### 2.2 工作目录

建议把工作目录设置到工具自身目录，至少要保证它和 `temp/` 的相对关系是稳定的。

原因很简单：

- 代码当前使用的是相对临时目录。
- 启动和退出时会清理这个目录。
- 如果工作目录漂移，临时文件位置会跟着漂移，排查时不直观。

### 2.3 单实例

如果启动器本身支持单实例控制，建议默认只保留一个工具实例。这个工具是围绕“当前帧列表”工作的，多开通常没有业务价值，反而容易让用户分不清哪个窗口在监听剪贴板。

## 3. 同步和升级

如果启动器会从远端同步脚本或版本，建议把以下内容当成同一发布单元：

- 可执行文件
- 运行入口配置
- 工作目录配置
- 版本说明

这样可以避免“入口更新了，但运行目录还是旧的”这种低级问题。

## 4. 排查建议

如果测试同事反馈“能打开，但行为不对”，优先检查这几项：

1. 是否启动到了旧版本。
2. 当前进程的工作目录是否正确。
3. `temp/` 是否被意外放到了别的目录。
4. 剪贴板是否真的是资源管理器的文件列表。
5. 目标机器是否还是 Windows 环境。

## 5. 对外口径

给测试同事的口径尽量保持简单：

- 复制图片
- 窗口里调顺序
- 点生成
- 去目标软件里粘贴

不要把内部的路径、临时目录和编译细节暴露给普通使用者。
